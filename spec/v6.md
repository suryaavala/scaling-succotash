# Project: Enterprise B2B Company Search - V6 End-to-End Demo

## 1. Project Context & Objectives
This specification outlines the creation of an interactive, end-to-end Jupyter Notebook (`notebooks/demo.ipynb`). The notebook will act as a deep-dive tutorial and showcase for the entire `scaling-succotash` platform. It will touch upon every core functionality path, providing clear illustrations and explanations for developers or stakeholders to understand the system's architecture and performance.

## 2. Notebook Structure

### Section 1: Introduction & Architecture Overview
*   **Explanation:** High-level summary of the dual-stage retrieval system (OpenSearch + Local ML inference), LLM Router heuristics, and Redis semantic caching.
*   **Action:** Print connection statuses to backing services (OpenSearch on 9200, Redis on 6379, Inference on 8001, FastAPI on 8000).

### Section 2: Data Exploration & Ingestion
*   **Explanation:** How raw company data is processed using `polars` and batched into OpenSearch.
*   **Action:** Load `data/companies.csv` via Polars, display a sample dataframe, and explain the `knn_vector` index schema mapping.

### Section 3: Deterministic Search (The Baseline)
*   **Explanation:** Executing rigid schema searches.
*   **Action:** Send a `POST` request to `/api/v2/search` applying specific industry/size filters. Display the formatted JSON response.

### Section 4: Intelligent Semantic Search
*   **Explanation:** How unstructured text is converted to dense vectors and matched against the OpenSearch `hnsw` index.
*   **Action:** Send a `POST` request to `/api/v2/search/intelligent` with a fuzzy query like "Cloud networking startups in California". Display the parsed LLM intent and resulting candidates.

### Section 5: The Agentic Workflow
*   **Explanation:** Routing complex, time-sensitive queries to the Agentic Strategy.
*   **Action:** Send a query like "Recent acquisitions by Cisco in the AI space". Explain how the router identifies `requires_agent: true`.

### Section 6: Performance Optimization & Caching Limits
*   **Explanation:** Demonstrating the V5 optimizations (Singleflight Request Collapsing & Redis Caching).
*   **Action:** Executing identical semantic queries sequentially. The notebook will measure and print the execution time difference (e.g., 1000ms vs 5ms) and assert the presence of the `X-Cache-Hit: true` HTTP response header.

## 3. Implementation Steps
1. Add `jupyter` to `pyproject.toml`.
2. Create `notebooks/` directory.
3. Generate `demo.ipynb` bridging all outlined sections with executeable Python code (`httpx`, `polars`, `time`).
